## 🎯 Проблематика и Решение

**Боль:** В крупных R&D и производственных центрах горно-металлургической отрасли ежедневно генерируются гигабайты технологических регламентов, научных отчетов и результатов экспериментов. Институциональная память теряется, а поиск неочевидных связей (какое оборудование подходит для конкретного сплава при определенной температуре) превращается в многочасовой ручной труд.

**Наше решение — AuraGraph RAG:**  
Мы разработали отказоустойчивую систему класса **Knowledge Graph RAG**. Это единый интерактивный граф знаний, который объединяет разрозненные тексты в четкую онтологию и использует ультрабыстрый семантический поиск с помощью LLM. Система за считанные секунды анализирует графовые связи между рудой, процессами и оборудованием, выдавая инженеру готовый структурированный ответ с визуальным пруфом.

## 🚀 Архитектура и SOTA-фичи

Наш пайплайн построен на принципах надежности и скорости:

* 🛡️ **Robust Extraction:** Продвинутый парсинг с умным алгоритмом возобновления (resume state). В случае обрыва соединения экстракция продолжится ровно с того же места, гарантируя отсутствие дублей и потери данных.
* 🧠 **Smart Entity Resolution:** SOTA-модуль очистки графа. Использование **Fuzzy Matching** (нечеткого поиска) с предварительной жесткой фильтрацией стоп-слов. Алгоритм идеально понимает русскую морфологию и падежи (успешно схлопывает «медной» и «медь»), отсекая информационный мусор.
* ⚡ **Ultra-Fast RAG (< 3 сек):** Интеграция с передовыми LPU-чипами от **Groq** (модель `llama-3.3-70b-versatile`). Ответ генерируется мгновенно, с полным попаданием в жесткие тайминги ТЗ. Граф преобразуется в плотный набор фактов, исключая превышение лимитов токенов (Ошибку 400).
* 🎨 **Interactive UI:** Современный пользовательский интерфейс на **Streamlit**. Вместо зависания браузера от попыток отрисовать десятки тысяч узлов, система извлекает локальный **Ego-graph** (подграф связей 1-го порядка) и рендерит только его через **Pyvis**.

## 📋 Онтология (Строго по ТЗ)

Наша база знаний опирается на строгую, выверенную онтологическую модель. Любые "галлюцинации" моделей жестко фильтруются на этапе очистки.

### Разрешенные Сущности (Nodes)
* 🟢 `Material` — Сырье, продукты, реактивы.
* 🟠 `Process` — Технологические и химические процессы.
* 🔵 `Equipment` — Установки, агрегаты, машины.
* 🟣 `Property` — Физико-химические свойства и метрики.
* 🔴 `Experiment` — Лабораторные и промышленные испытания.
* ⚫ `Publication` — Источники, регламенты, статьи.
* циановый `Expert` — Авторы, технологи, исследователи.
* 🟡 `Facility` — Цеха, заводы, месторождения.

### Разрешенные Связи (Edges)
* 🔗 `uses_material`
* 🔗 `operates_at_condition`
* 🔗 `produces_output`
* 🔗 `described_in`
* 🔗 `validated_by`
* 🔗 `contradicts`

## ⚙️ Быстрый старт (Local Setup)

Проект готов к развертыванию на любой локальной машине. 

**1. Клонирование и установка зависимостей**
```bash
git clone https://github.com/your-repo/nornickel-ai-hackathon.git
cd nornickel-ai-hackathon
pip install streamlit pyvis networkx groq pydantic tqdm requests tenacity
```

**2. Запуск пайплайна очистки (Entity Resolution)**
Этот шаг нормализует сырые триплеты, применяет строгую онтологию и убирает мусор.
```bash
python src/entity_resolution/clean_triplets.py
```

**3. Запуск веб-интерфейса**
```bash
streamlit run app/ui.py
```

## 💡 Демонстрация работы

![Скриншот интерфейса - Главный экран](link_to_screenshot_1.png)

В интерфейсе просто введите свой вопрос. Ассистент поймет суть, локализует подграф и выдаст точный ответ.

**Пример убойного запроса:**
> *"Какие процессы и оборудование подходят для переработки медной руды?"*

**Результат:**
Система мгновенно распознает сущность `Медная руда`, извлекает все связанные с ней процессы (`Флотация`, `Плавка`) и агрегаты (`Флотационная машина`, `Печь Ванюкова`), генерирует текстовый саммари и отрисовывает аккуратный граф в качестве пруфа!
